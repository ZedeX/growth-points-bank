## 1. 产品概述

### 1.1 产品名称

暑假成长积分银行（Summer Growth Points Bank）

### 1.2 产品定位

面向家庭场景的成长激励 Web 应用，将儿童暑假期间的五大成长维度（学习力、运动力、自控力、探索力、实践力）通过积分银行机制游戏化，用代币经济催化孩子的内在驱动力。

### 1.3 核心问题

宏大的成长目标与孩子缺乏即时正反馈之间的矛盾。传统家长式管理（"你要乖"、"好好看书"）让孩子迷茫，无法将大目标拆解为可执行的日常动作。

### 1.4 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|---------|
| 家长（主用户） | 有学龄期孩子的家长，关注孩子暑假成长 | 提供结构化的成长框架，减少催促和冲突 |
| 孩子（端用户） | 6-14岁学龄儿童，暑假期间需要日常安排 | 游戏化体验，即时正反馈，自主选择奖励 |

### 1.5 核心理念

**将无形的"自律"转变为有形的"资产"，用代币机制催化孩子的自发驱动力。**

孩子从"被管理者"变为时间的"主导者"——当他们为筹集"一次短途旅行"或"一次家庭电影"的积分而主动练习、运动、分担家务时，成长就自然发生了。

### 1.6 技术平台

移动端优先的 H5 Web 应用，适配手机和平板浏览器。

## 2. 用户角色与权限

### 2.1 角色1：家长（Parent）

**职责**：
- 注册/登录家庭账户
- 创建和管理多个孩子档案（姓名、头像、年龄段）
- 配置任务：为每个成长维度创建/编辑/删除任务，设定积分值
- 配置奖励：自定义三个档位的奖励内容
- 参与每周复盘：填写"家长看见的进步"
- 查看数据：孩子积分明细、任务完成率、维度点亮情况、成长日记
- 审核兑换：确认孩子的奖励兑换请求

### 2.2 角色2：孩子（Child）

**职责**：
- 通过家长分享的链接/二维码进入（无需独立注册）
- 查看成长地图：五大维度的关卡进度和点亮状态
- 每日打卡：按分类筛选任务，勾选完成，实时获得积分
- 查看积分余额和历史记录
- 兑换奖励：浏览三档奖励，发起兑换请求
- 记录成长日记：用文字记录成长点滴
- 参与每周复盘：填写"最棒的一件事"和"遇到的困难"

### 2.3 权限矩阵

| 操作 | 家长 | 孩子 |
|------|:----:|:----:|
| 管理家庭账户 | ✅ | ❌ |
| 管理孩子档案 | ✅ | ❌ |
| 配置任务和积分 | ✅ | ❌ |
| 配置奖励内容 | ✅ | ❌ |
| 每日打卡 | ❌ | ✅ |
| 查看自己的积分 | ✅（查看孩子） | ✅ |
| 发起奖励兑换 | ❌ | ✅ |
| 审核兑换请求 | ✅ | ❌ |
| 记录成长日记 | ✅（代记） | ✅ |
| 每周复盘填写 | ✅ | ✅ |
| 查看复盘记录 | ✅ | ✅（仅自己） |
| 查看数据看板 | ✅ | ❌ |

## 3. 核心功能详述

### 3.1 成长地图（Growth Map）

**功能定位**：应用首页，五大成长维度以"闯关关卡"形式可视化呈现。

**五大维度定义**：

| 维度 | 说明 | 典型任务示例 |
|------|------|-------------|
| 学习力 | 阅读、练字、复盘，让学习变成稳定的好习惯 | 写日记、阅读30分钟、练字一页 |
| 运动力 | 运动和户外，让身体充满假期的电量 | 户外活动2小时、跳绳100下、游泳 |
| 自控力 | 安排时间、管理屏幕，把计划握在自己手里 | 屏幕时间<2小时、按时起床、自主安排时间表 |
| 探索力 | 兴趣、情绪和沟通，找到属于自己的好奇心 | 尝试新事物、情绪日记、主动沟通 |
| 实践力 | 家务、做饭、志愿服务，把成长做出来 | 洗碗、做一道菜、社区志愿服务 |

**页面元素**：
- 顶部进度条：今日已完成任务数 / 今日总任务数
- 五个维度关卡卡片，纵向排列，每张卡片包含：
  - 维度图标（emoji 或矢量图标）
  - 维度名称（彩色文字）
  - 进度指示：当前点亮星星数 / 总星星数
  - 维度状态色：未开始（灰色 #9E9E9E）→ 进行中（部分点亮 #FFC107）→ 已完成（全亮金色 #FFD700）
- 底部快捷入口：今日打卡按钮

**交互逻辑**：
1. 点击任一维度卡片 → 展开该维度的专属任务列表（折叠面板）
2. 任务列表中可直接勾选完成 → 卡片星星点亮动画
3. 完成当日该维度全部任务 → 关卡被"正式点亮"，星星全部亮起
4. 返回地图视图 → 已完成维度卡片显示金色发光效果

**状态规则**：
- 每日 00:00 重置当日任务完成状态（积分保留）
- 维度点亮状态按日计算，每日独立
- 历史点亮记录可在数据看板查看

### 3.2 今日打卡（Daily Check-in）

**功能定位**：孩子每天的"赚积分"现场，展示当日所有待完成任务。

**页面元素**：
- 顶部筛选标签栏（横向滚动）：全部 / 学习力 / 运动力 / 自控力 / 探索力 / 实践力
- 任务卡片列表，每张卡片包含：
  - 任务名称
  - 所属维度彩色标签（学习力-蓝、运动力-橙、自控力-紫、探索力-绿、实践力-红）
  - 积分值（如 "+2分"）
  - 完成状态：未完成（空心圆圈）/ 已完成（绿色勾选 + 灰色卡片）
- 底部固定统计栏：今日已完成 X 项任务，获得 Y 积分
- 积分动画：打卡时积分数字从底部向上飘浮 "+2" 动效

**交互逻辑**：
1. 点击任务卡片的勾选按钮 → 标记完成
2. 积分实时入账 → 积分数字动画上升 → 卡片变为已完成状态
3. 已完成任务可点击取消勾选 → 扣除已得积分
4. 打卡仅限当日（00:00-23:59），不可补打卡
5. 筛选标签点击 -> 仅显示该维度的任务

**任务验证机制（信任+抽检）**：
- 默认信任孩子自我打卡，保持低摩擦体验
- 家长端每日显示"今日打卡清单"，家长可查看但不需要逐项审批
- 家长可"撤销"某项不实打卡（扣除对应积分）
- 数据看板中显示"家长撤销次数"，作为诚信参考指标

**积分规则**：
- 每个任务有固定积分值（由家长设定）
- 完成任务立即获得积分
- 取消打卡立即扣除对应积分
- 每日任务每日 00:00 重置（频率为 daily 的任务）
- 每周任务每周一 00:00 重置（频率为 weekly 的任务）
- 一次性任务完成后永久消失（频率为 once 的任务）

**示例任务与积分**（来自原文）：
- 认真写日记 → 学习力 +2分
- 户外活动2小时 → 运动力 +2分
- 控制屏幕时间不超过2小时 → 自控力 +2分

### 3.3 每周家庭复盘（Weekly Family Review）

**功能定位**：每周一次的家庭复盘对话工具，5~10分钟，让孩子反思本周表现，实现双向沟通。

**页面结构**：

**Part A - 孩子填写区**：
- "这周我最棒的一件事是……"（多行文本输入，建议50-200字）
- "这周我遇到的困难是……"（多行文本输入，建议50-200字）

**Part B - 双向沟通区**：
- 【家长看见的进步】：家长填写，孩子可见（多行文本输入）
- 【孩子希望的支持】：孩子填写，家长可见（多行文本输入）

**Part C - 本周数据汇总**（自动统计，只读）：
- 完成任务数：X 项
- 存入积分：Y 分
- 点亮维度数：Z 个

**交互逻辑**：
1. 每周日 18:00 弹出复盘提醒（可在设置中修改提醒时间）
2. 孩子和家长可分别填写各自区域，互不阻塞
3. 所有人填写完成后 -> 生成本周复盘记录
4. 历史复盘记录按周列表查看，可展开详情
5. 复盘记录不可编辑（提交后只读）

**双盲机制**：
- 双方在未提交前，互相看不到对方的填写内容
- 任意一方先提交后，可以看到对方已提交的内容
- 如果对方尚未提交，显示"等待对方填写中..."
- 双方都提交后，复盘记录锁定为只读

**设计意图**：
这是整个系统中最得意的功能——让困难被说出来，让改变每周发生。这是真正的"脚手架"，而非简单的打卡工具。

### 3.4 自定义奖励兑换（Custom Reward Redemption）

**功能定位**：积分兑换奖励的商城，三个积分档位，奖励内容家长完全自定义。

**三个档位**：

| 档位 | 积分门槛 | 示例奖励 |
|------|---------|---------|
| 小奖励 | 10分 | 选择一次家庭电影、点播一次睡前故事、选择一次晚餐菜单 |
| 中奖励 | 30分 | 买一本喜欢的书、一次朋友聚会、一次亲子外出 |
| 大奖励 | 60分 | 一次短途旅行、实现一个孩子期待的小愿望 |

**设计原则**：
- 排除冰冷的物质奖励为主，侧重体验型奖励
- 所有奖励内容家长可完全自定义
- 引导孩子思考努力与奖励的关联

**页面元素**：
- 积分余额显示（顶部）
- 三档奖励分区展示，每个奖励卡片包含：
  - 奖励标题
  - 积分价格
  - 奖励描述
  - "兑换"按钮（积分不足时置灰，显示"差 X 分"）
- 兑换历史记录入口

**兑换流程**：
1. 孩子浏览奖励列表 -> 选择想要兑换的奖励
2. 系统弹出引导提示："这个奖励对应了你哪一段坚持？"
3. 孩子填写简短说明（文本输入，建议20-100字）
4. 发起兑换请求 -> 状态为"待审核"
5. 家长收到通知 -> 审核确认（通过/拒绝）
6. 通过 -> 扣除相应积分 -> 状态变为"待履约"
7. 家长实际兑现后，点击"标记已兑现" -> 状态变为"已兑现"
8. 拒绝 -> 积分不扣除 -> 附带拒绝说明

**履约追踪**：
- 超过7天未标记兑现，系统提醒家长
- 孩子端可看到"待履约"状态
- 状态流转：pending(待审核) -> approved(已通过/待履约) -> fulfilled(已兑现) / rejected(已拒绝)

**家长管理**：
- 编辑每个档位的奖励内容
- 新增/删除奖励项
- 修改奖励积分价格（需大于0）
- 查看兑换历史
- 审核待处理的兑换请求

**核心理念**：
> "积分不是为了控制孩子，而是为了让孩子看见：我的努力是有价值的。"
> "兑换前，可以让孩子说一说'这个奖励对应了我哪一段坚持'。"

### 3.5 成长日记（Growth Diary）

**功能定位**：文字版的成长记录工具，让孩子用文字记录暑假期间的成长点滴，轻量、低门槛，不消耗云端存储资源。

**日记分类**：
画画记录 / 日记 / 做饭记录 / 实验记录 / 运动记录 / 其他

**每条日记包含**：
- 日期
- 标题
- 内容（多行文本）
- 分类

**页面元素**：
- 顶部统计：日记总数
- 分类筛选标签（横向滚动）
- 时间线视图：按日期倒序排列所有日记
- 每条日记卡片包含：标题、分类标签、内容预览、日期
- 浮动新建按钮：写日记

**新建流程**：
1. 点击新建按钮 -> 进入日记编辑页
2. 填写标题（必填）
3. 填写日记内容（多行文本，必填）
4. 选择分类标签
5. 点击保存 -> 新日记出现在时间线顶部

**暑假结束自动生成**：
- 个人暑假成长档案（文字版）
- 汇总内容：日记精选亮点、积分总览、各维度成长数据、每周复盘精华
- 可分享/导出

## 4. 补充功能

### 4.1 用户认证与家庭账户

**家长注册与登录**：
- 注册方式：手机号或邮箱
- 登录方式：密码登录
- 一个家长账户对应一个家庭

**孩子访问机制**：
- 家长在孩子档案页生成专属访问链接/二维码
- 孩子扫码/点击链接 → 直接进入自己的成长地图，无需注册
- 孩子端 Token 有效期 7 天，过期需重新扫码
- 一个链接对应一个孩子，不可混用

### 4.2 多孩子管理

- 家长可创建多个孩子档案
- 每个孩子档案包含：姓名、头像、年龄段（6-8岁/9-11岁/12-14岁）
- 每个孩子独立拥有：积分账户、任务进度、成长地图、成长日记、复盘记录
- 家长端可切换查看不同孩子的数据
- 孩子端仅看到自己的数据

### 4.3 任务管理（家长端）

**任务创建**：
- 任务名称（必填，1-30字）
- 所属维度（必选，五选一）
- 积分值（必填，1-20分）
- 频率：每日（daily）/ 每周（weekly）/ 一次性（once）
- 任务描述（选填）

**任务模板**：
- 预设常见任务模板供快速创建
- 按维度分类的模板库
- 家长可基于模板修改后创建

**任务管理**：
- 编辑已有任务
- 删除任务（已打卡记录保留）
- 启用/停用任务（停用后不在打卡列表显示）

### 4.4 积分系统

**积分规则**：
- 积分来源：仅通过完成任务获得
- 积分消耗：仅通过兑换奖励消耗
- 积分不可转让（孩子间不可互转）
- 积分不可手动调整（家长不可直接增减）
- 积分永久保留，跨暑假累积，不因暑假结束而清零
- 孩子可跨暑假储蓄积分以兑换大额奖励
- 成长日记和复盘记录按暑假周期归档

**积分明细流水**：
- 时间戳
- 来源类型：任务完成 / 奖励兑换
- 来源详情：任务名称 / 奖励名称
- 变动值：+N / -N
- 操作后余额

**汇总统计**：
- 按维度统计积分分布
- 按周/月统计积分趋势
- 总积分、已使用积分、可用积分

### 4.5 设置与个性化

- 暑假起止日期设置（影响成长档案生成时间）
- 孩子档案管理（增删改）
- 每周复盘提醒时间设置
- 主色调：清新绿（#4CAF50 系列）
- 家庭成就墙开关（开启后孩子可看到兄弟姐妹的维度点亮状态，但不显示积分数字）
- 任务模板按年龄段推荐（6-8岁/9-11岁/12-14岁不同模板）
- 关于与反馈

## 5. 数据模型

### 5.1 实体关系图

```
Family 1───n Child
Family 1───1 Parent
Family 1───n Task
Family 1───n Reward
Child 1───n CheckIn
Child 1───n PointTransaction
Child 1───n RewardRedemption
Child 1───n WeeklyReview
Child 1───n GrowthDiary
Task n───1 Dimension
```

### 5.2 实体定义

#### Family（家庭）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| parent_id | UUID | 关联家长 |
| name | String | 家庭名称 |
| summer_start_date | Date | 暑假开始日期 |
| summer_end_date | Date | 暑假结束日期 |
| created_at | DateTime | 创建时间 |

#### Parent（家长）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| email | String | 邮箱（可选） |
| phone | String | 手机号（可选） |
| password_hash | String | 密码哈希 |
| nickname | String | 昵称 |
| created_at | DateTime | 创建时间 |

#### Child（孩子）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| family_id | UUID | 关联家庭 |
| name | String | 姓名 |
| avatar | String | 头像URL |
| age_group | Enum | 年龄段：6-8/9-11/12-14 |
| access_token | String | 孩子端访问令牌 |
| token_expires_at | DateTime | 令牌过期时间 |
| created_at | DateTime | 创建时间 |

## 8. 设计理念与心理学基础

### 8.1 核心心理学原理

**代币经济（Token Economy）**：
将无形的"自律"转变为有形的"资产"。通过积分作为中介，将抽象的成长目标具象化为可量化、可累积的"货币"，让孩子直观感受到努力的价值。

**即时正反馈**：
传统的成长要求（"你要乖"）反馈周期长、标准模糊。积分银行将反馈缩短到"勾选即可看到积分入账"的秒级，满足孩子对即时反馈的心理需求。

**内在驱动力催化**：
积分不是终点，而是桥梁。当孩子为筹集"一次短途旅行"而主动练习、运动、分担家务时，他们从"被管理者"变成了自己时间的"主导者"。

### 8.2 五大维度的设计逻辑

五大维度并非随意划分，而是对应儿童全面发展的核心能力：
- **学习力**：认知发展，养成稳定的学习习惯
- **运动力**：身体发展，保持健康体魄
- **自控力**：执行功能，学会自我管理
- **探索力**：情感与社交发展，保持好奇心
- **实践力**：生活技能，将成长落地为行动

### 8.3 三大闭环设计

1. **每日闭环**：打卡 → 积分入账 → 成长地图点亮（即时反馈）
2. **每周闭环**：复盘 → 发现困难 → 调整节奏（中期修正）
3. **暑假闭环**：积累积分 → 兑换奖励 → 生成成长档案（长期目标）

### 8.4 奖励设计哲学

> "积分不是为了控制孩子，而是为了让孩子看见：我的努力是有价值的。"

- 排除冰冷的物质奖励为主，侧重体验型和家庭互动型奖励
- 兑换前引导孩子思考"这个奖励对应了我哪一段坚持"
- 家长完全自定义，适配不同家庭的价值观

## 9. Grilling决策记录

以下决策通过深度拷问（Grilling）流程确定：

| # | 决策项 | 决定 | 理由 |
|---|--------|------|------|
| 1 | 任务验证机制 | 信任+抽检 | 保持低摩擦，家长可撤销不实打卡 |
| 2 | 积分过期策略 | 永久保留 | 跨暑假累积，支持长期储蓄目标 |
| 3 | 兄弟姐妹可见性 | 默认不可见+成就墙 | 避免不健康竞争，保留共同成长氛围 |
| 4 | 每周复盘双盲 | 真正双盲 | 防止互相引导，保证反馈真实性 |
| 5 | 惩罚机制 | 无惩罚 | 与正向激励理念一致，不设扣分 |
| 6 | 奖励履约追踪 | 增加追踪 | 待履约->已兑现，7天提醒，防止遗忘 |
| 7 | 年龄适配 | 模板适配 | 任务模板按年龄分组，核心功能不变 |
| 8 | 离线冲突处理 | 最后写入+告警 | 家长撤销权限优先，冲突时家长端弹提示 |
| 9 | 数据导出 | 完整导出 | PDF成长档案+日记导出，注销后30天删除 |
| 10 | MVP范围 | 分两期 | 第一期核心闭环，第二期深化功能 |
