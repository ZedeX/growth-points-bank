fix(auth): make all require* preHandlers async to fix Fastify hang

ROOT CAUSE FOUND: Fastify 4 requires preHandler functions to be async
(return Promise) or use callback style. Synchronous preHandler functions
that return void cause Fastify to hang indefinitely waiting for a
non-existent Promise resolution.

This was the actual root cause of ALL CI test timeouts - not jose, not
pool.on('connect'), not PostgreSQL. The sync requireParent/requireChild/
requireAuth/requireFamilyId functions caused every authenticated request
to hang.

Fix:
- All require* functions now return Promise<void>
- requireFamilyId now returns Promise<string | null>
- All 25 call sites updated with `await`
- authMiddleware restored to async (not callback style)
- Removed all diagnostic console.log
- Removed jose dependency entirely (using native crypto)
- Fixed createChildSchema avatar to accept null (nullish)
- Added ADR-0010 for upcoming SQLite migration
- Removed debug scripts

Local test results: 59 passed | 29 failed (assertion mismatches, not timeouts)
Duration: 7 seconds (was 105+ seconds with timeouts)
